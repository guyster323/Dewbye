# 기상 데이터 기반 HVAC 운용 분석 플랫폼 - 기본 설계

**프로젝트명**: Weather-HVAC Analytics - 건물 공조 시스템 운용 최적화 분석 도구  
**타겟 사용자**: 건물 에너지 관리자, BEMS 운영사, 건설사 시설팀  
**핵심 가치**: 기상 데이터와 HVAC 동작 패턴의 상관관계 분석 → 건물 성능 최적화 기초 데이터 제공

---

## 1. 개념 & 문제 정의

### 1.1 배경

**BEMS (Building Energy Management System) 운영의 현실**
- 건물의 난방/냉방/환기 에너지는 전체 소비량의 40~60% 차지
- 기존 접근: 설비 교체 중심 (초기 투자 높음, 회수기간 길음)
- **미해결 문제**: 기상 변화에 따른 HVAC 실시간 최적 운용 전략 부재

### 1.2 당신의 핵심 인사이트

> **HVAC 송풍모드 전환 시점**이 건물의 온습도 급격한 변화를 만들고,
> 이것이 누전, 곰팡이, 건물 부식 등 다양한 하자와 연관될 수 있다는 가설

**추적할 패턴**:
1. **외기 온도 변동** → HVAC 냉방/난방 모드 전환 시점 예측
2. **실내 상대습도 급변화** → 제습 능력의 한계점 감지
3. **이슬점과 실내 온도의 접근** → 결로 위험 신호
4. **건물별 HVAC 응답 지연** → 각 건물의 고유한 열특성 분석

### 1.3 사용 사례

**Case 1: 건물 에너지 관리자**
```
5월 20일 오후 3시경
• 외기 21°C → 24°C 급상승
• HVAC 냉방으로 전환
• 실내 습도 15% 급상승
  (외기 습도 70% + 냉방 능력 한계)
• 확인: 지하 배관실 결로 발생
→ 제습기 추가 운전 결정
```

**Case 2: 건설사 준공검수**
```
신축 건물 준공 후
• 6개월 기상 데이터 분석
• 습도 급변 구간 파악
• 해당 시간대 누전 발생 기록 대조
• "이 시간대 배선 점검 필요" → 하자 사전 예방
```

**Case 3: 주택 소유자**
```
누전 차단기 자주 내려감
• 앱으로 분석
• 매년 6월 15일~22일 높은 위험도 패턴 발견
• 그 기간에 제습기 집중 운용
• 누전 횟수 50% 감소
```

### 2.2 핵심 계산 공식

#### 절대습도 (Absolute Humidity, AH)
$$AH = \frac{RH \times e^s}{461.5 \times (T + 273.15)}$$

**의미**: 공기 1m³에 포함된 수증기의 실제 질량 (g/m³)

#### 포화 수증기압 (Magnus 공식)
$$e^s = 6.1094 \times e^{\left(\frac{17.625 \times T}{T + 243.04}\right)}$$

#### 이슬점 (Dew Point, Td)
$$Td = \frac{243.04 \times \ln(RH/100 \times e^s / 6.1094)}{17.625 - \ln(RH/100 \times e^s / 6.1094)}$$

#### 온도 변화율
$$\frac{dT}{dt} = \frac{T_{t+1} - T_t}{\Delta t} \quad (℃/hour)$$
**임계값**: `|dT/dt| > 1.5℃/hour` → HVAC 모드 전환 신호

#### 습도 변화율
$$\frac{dRH}{dt} = \frac{RH_{t+1} - RH_t}{\Delta t} \quad (\%/hour)$$
**임계값**: `dRH/dt > 2%/hour` → 비정상 습도 상승
